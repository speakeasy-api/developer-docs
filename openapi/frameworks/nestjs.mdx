# How to generate an OpenAPI document with NestJS

Generating an OpenAPI document in NestJS is often less work than it first appears. Controllers, DTOs, validation decorators, and response models already describe a large part of the API contract, which means much of the structure needed for OpenAPI is already present in the application code.

This tutorial shows how to turn that existing structure into a useful OpenAPI document and then use it with Speakeasy to generate SDKs. The example in this guide uses a train travel API domain with stations, trips, bookings, and payments, with a focus on keeping the generated document clear enough for both reference docs and SDK generation.

<Callout title="Example code" type="info">
  Follow this guide with the NestJS example in this repository at
  <code>examples/frameworks-nestjs</code>.
</Callout>

Here is what this guide covers:

1. Install and configure `@nestjs/swagger` and Scalar API Reference.
2. Generate an OpenAPI 3.2 document from NestJS controllers and DTOs.
3. Add stable `operationId`, tags, and component schemas for SDK quality.
4. Add security schemes and Speakeasy extensions.
5. Write `openapi.yaml` and generate SDKs from it.

## Requirements

This guide assumes a NestJS application and basic familiarity with controllers, DTOs, and decorators.

Most of the work in this flow happens by adding precise metadata to code that already exists, rather than maintaining a separate OpenAPI file by hand.

Install Node.js and the NestJS CLI:

```bash filename="Terminal"
npm install -g @nestjs/cli
```

Verify the CLI:

```bash filename="Terminal"

```


## Add Swagger to NestJS

Install the OpenAPI and docs dependencies:

```bash filename="Terminal"
npm install @nestjs/swagger @scalar/nestjs-api-reference
```

Configure Swagger and Scalar in `src/main.ts`.

Start with the base document metadata:

```typescript filename="src/main.ts"
async function bootstrap() {
  const app = await NestFactory.create(AppModule);

  const config = new DocumentBuilder()
    .setOpenAPIVersion('3.2.0')
    .setTitle('Train Travel API')
    .setDescription('Find stations, discover trips, and create bookings across Europe.')
    .setVersion('1.2.1')
    .addServer('https://api.example.com', 'Production server')
    .addServer('http://localhost:3000', 'Local development server')
    .build();
}
```

NestJS Swagger supports OpenAPI 3.2 directly through `DocumentBuilder.setOpenAPIVersion()`.

Add Scalar and write the generated document to `openapi.yaml`:

```typescript filename="src/main.ts"
import { ValidationPipe } from '@nestjs/common';
import { NestFactory } from '@nestjs/core';
import { writeFileSync } from 'fs';
import * as yaml from 'js-yaml';
import { apiReference } from '@scalar/nestjs-api-reference';
import {
  DocumentBuilder,
  SwaggerDocumentOptions,
  SwaggerModule,
} from '@nestjs/swagger';
import { AppModule } from './app.module';

async function bootstrap() {
  const app = await NestFactory.create(AppModule);

  app.useGlobalPipes(
    new ValidationPipe({
      whitelist: true,
      forbidNonWhitelisted: true,
      transform: true,
      errorHttpStatusCode: 400,
    }),
  );

  const config = new DocumentBuilder()
    .setOpenAPIVersion('3.2.0')
    .setTitle('Train Travel API')
    .setDescription('Find stations, discover trips, and create bookings across Europe.')
    .setVersion('1.2.1')
    .addTag('Resources', 'Top-level navigation tag for related resources')
    .addTag('Stations', 'Find and filter train stations')
    .addTag('Trips', 'Timetables and routes for train trips')
    .addTag('Bookings', 'Create and manage trip bookings')
    .addServer('https://api.example.com', 'Production server')
    .addServer('http://localhost:3000', 'Local development server')
    .addBearerAuth(
      {
        type: 'http',
        scheme: 'bearer',
        bearerFormat: 'JWT',
        description: 'Bearer token authentication',
      },
      'BearerAuth',
    )
    .addOAuth2(
      {
        type: 'oauth2',
        flows: {
          authorizationCode: {
            authorizationUrl: 'https://auth.example.com/oauth2/authorize',
            tokenUrl: 'https://auth.example.com/oauth2/token',
            scopes: {
              'bookings:read': 'Read bookings',
              'bookings:write': 'Create and update bookings',
            },
          },
        },
      },
      'OAuth2',
    )
    .build();

  const options: SwaggerDocumentOptions = {
    operationIdFactory: (_controllerKey: string, methodKey: string) => methodKey,
  };

  const document = SwaggerModule.createDocument(app, config, options);

  writeFileSync('openapi.yaml', yaml.dump(document, { noRefs: true }));

  SwaggerModule.setup('api-json', app, document);
  app.use('/api', apiReference({ spec: { content: document } }));

  await app.listen(3000);
}

bootstrap();
```

## Describe routes with OpenAPI decorators

NestJS route decorators define runtime behavior. OpenAPI decorators add the details needed for generated docs and SDKs.

Start small and build outward. A single well-described collection route is enough to verify that tags, parameters, and response schemas are being emitted the way the document needs them.

Start with a simple collection route:

```typescript filename="src/train/train.controller.ts"
import { Controller, Get, Query } from '@nestjs/common';
import { ApiOkResponse, ApiOperation, ApiQuery, ApiTags } from '@nestjs/swagger';

@ApiTags('Resources')
@Controller()
export class TrainController {
  @Get('stations')
  @ApiTags('Stations')
  @ApiOperation({ summary: 'Get a list of train stations' })
  @ApiQuery({ name: 'search', required: false, type: String })
  @ApiQuery({ name: 'country', required: false, type: String })
  @ApiOkResponse({ type: [Station] })
  getStations(@Query('search') search?: string, @Query('country') country?: string) {}
}
```

Add DTO-backed query parameters for routes like `GET /trips`:

```typescript filename="src/train/dto/trips-query.dto.ts"
import { IsBoolean, IsDateString, IsOptional, IsUUID } from 'class-validator';

export class TripsQueryDto {
  /**
   * Origin station ID.
   * @example 'efdbb9d1-02c2-4bc3-afb7-6788d8782b1e'
   */
  @IsUUID()
  readonly origin!: string;

  /**
   * Destination station ID.
   * @example 'b2e783e1-c824-4d63-b37a-d8d698862f1d'
   */
  @IsUUID()
  readonly destination!: string;

  /**
   * Desired departure date in ISO 8601.
   * @example '2026-05-21T09:00:00Z'
   */
  @IsDateString()
  readonly date!: string;

  @IsOptional()
  @IsBoolean()
  readonly bicycles?: boolean;

  @IsOptional()
  @IsBoolean()
  readonly dogs?: boolean;
}
```

Then use it in the controller:

```typescript filename="src/train/train.controller.ts"
@Get('trips')
@ApiTags('Trips')
@ApiOperation({ summary: 'Get available train trips' })
@ApiOkResponse({ type: [Trip] })
getTrips(@Query() query: TripsQueryDto): Trip[] {
  return this.trainService.getTrips(query);
}
```

## Ensure schemas include property metadata

Without the Swagger plugin, generated component schemas can stay too thin for high-quality docs or SDKs. Enabling comment introspection fills in descriptions and examples from the model definitions already used by the app.

Enable the Nest Swagger plugin in `nest-cli.json`:

```json filename="nest-cli.json"
{
  "$schema": "https://json.schemastore.org/nest-cli",
  "collection": "@nestjs/schematics",
  "sourceRoot": "src",
  "compilerOptions": {
    "deleteOutDir": true,
    "plugins": [
      {
        "name": "@nestjs/swagger",
        "options": {
          "introspectComments": true
        }
      }
    ]
  }
}
```

With comment introspection enabled, doc comments on DTO and entity fields become descriptions and examples in `components.schemas`.

Example entity:

```typescript filename="src/train/entities/station.entity.ts"
import { IsString, IsUUID } from 'class-validator';

export class Station {
  /**
   * Unique identifier for the station.
   * @example 'efdbb9d1-02c2-4bc3-afb7-6788d8782b1e'
   */
  @IsUUID()
  readonly id!: string;

  /**
   * Human-readable station name.
   * @example 'Berlin Hauptbahnhof'
   */
  @IsString()
  readonly name!: string;
}
```

This produces component schemas with stable names and useful descriptions instead of empty object definitions.

## Add stable `operationId` values and tags

SDK method names are derived from `operationId`, and SDK groupings are derived from tags.

Default operation names can become verbose, especially once controller names are included. Tightening them here keeps generated SDK method names shorter and easier to scan.

Use `SwaggerDocumentOptions` to keep `operationId` values short and stable:

```typescript filename="src/main.ts"
const options: SwaggerDocumentOptions = {
  operationIdFactory: (_controllerKey: string, methodKey: string) => methodKey,
};

const document = SwaggerModule.createDocument(app, config, options);
```

This keeps operation names like `getStations`, `getTrips`, and `createBooking` instead of controller-prefixed names.

Use route tags to keep generated docs and SDK groups readable:

```typescript filename="src/train/train.controller.ts"
@ApiTags('Resources')
@Controller()
export class TrainController {
  @Get('bookings')
  @ApiTags('Bookings')
  @ApiOperation({ summary: 'List existing bookings' })
  @ApiOkResponse({ type: [Booking] })
  listBookings(): Booking[] {
    return this.trainService.listBookings();
  }
}
```

## Add security and Speakeasy retries

After the basic route and schema structure is in place, add the parts that most directly affect SDK behavior: authentication and retry behavior.

Define security schemes in `DocumentBuilder`:

```typescript filename="src/main.ts"
const config = new DocumentBuilder()
  .addBearerAuth(
    {
      type: 'http',
      scheme: 'bearer',
      bearerFormat: 'JWT',
      description: 'Bearer token authentication',
    },
    'BearerAuth',
  )
  .addOAuth2(
    {
      type: 'oauth2',
      flows: {
        authorizationCode: {
          authorizationUrl: 'https://auth.example.com/oauth2/authorize',
          tokenUrl: 'https://auth.example.com/oauth2/token',
          scopes: {
            'bookings:read': 'Read bookings',
            'bookings:write': 'Create and update bookings',
          },
        },
      },
    },
    'OAuth2',
  )
  .build();
```

Apply those schemes at the controller level:

```typescript filename="src/train/train.controller.ts"
import { ApiBearerAuth, ApiOAuth2 } from '@nestjs/swagger';

@ApiBearerAuth('BearerAuth')
@ApiOAuth2(['bookings:read', 'bookings:write'], 'OAuth2')
@Controller()
export class TrainController {}
```

Add Speakeasy retry behavior globally:

```typescript filename="src/main.ts"
const config = new DocumentBuilder()
  .addExtension('x-speakeasy-retries', {
    strategy: 'backoff',
    backoff: {
      initialInterval: 500,
      maxInterval: 60000,
      maxElapsedTime: 3600000,
      exponent: 1.5,
    },
    statusCodes: ['5XX'],
    retryConnectionErrors: true,
  })
  .build();
```

Override retries for one operation when needed:

```typescript filename="src/train/train.controller.ts"
@Post('bookings')
@ApiTags('Bookings')
@ApiOperation({ summary: 'Create a booking' })
@ApiBody({ type: CreateBookingDto })
@ApiCreatedResponse({ type: Booking })
@ApiExtension('x-speakeasy-retries', {
  strategy: 'backoff',
  backoff: {
    initialInterval: 1000,
    maxInterval: 80000,
    maxElapsedTime: 3600000,
    exponent: 1.5,
  },
  statusCodes: ['5XX'],
  retryConnectionErrors: true,
})
createBooking(@Body() payload: CreateBookingDto): Booking {
  return this.trainService.createBooking(payload);
}
```

## Save the document and generate SDKs

In this example, `package.json` includes:

```json filename="package.json"
{
  "scripts": {
    "build": "nest build",
    "generate:openapi": "npm run build && GENERATE_OPENAPI_ONLY=true node dist/main.js"
  }
}
```

Generate OpenAPI:

```bash filename="Terminal"
npm run generate:openapi
```

The generated file contains OpenAPI 3.2 metadata, route parameters derived from DTOs, security requirements, and component schemas generated from entities.

Example output:

```yaml filename="openapi.yaml"
openapi: 3.2.0

paths:
  /stations:
    get:
      operationId: getStations
      tags:
        - Resources
        - Stations

  /trips:
    get:
      operationId: getTrips
      parameters:
        - name: origin
          required: true
          in: query
          schema:
            type: string
            format: uuid

components:
  securitySchemes:
    OAuth2:
      type: oauth2
      flows:
        authorizationCode:
          authorizationUrl: https://auth.example.com/oauth2/authorize
          tokenUrl: https://auth.example.com/oauth2/token
          scopes:
            'bookings:read': Read bookings
            'bookings:write': Create and update bookings

  schemas:
    Station:
      type: object
      properties:
        id:
          type: string
          format: uuid
        name:
          type: string
```

Generate an SDK:

```bash filename="Terminal"
speakeasy quickstart
```

Or run Speakeasy directly:

```bash filename="Terminal"
speakeasy generate sdk --schema ./openapi.yaml --lang typescript --out ./sdk-typescript
```

## Validate the document

Lint the generated OpenAPI file:

```bash filename="Terminal"
speakeasy lint openapi --schema ./openapi.yaml
```

## Example SDK usage

After generating a TypeScript SDK, call operations using methods named from `operationId` values.

```typescript filename="sdk-usage.ts"
import { SDK } from './sdk-typescript/src';
import { getStations, createBooking } from './sdk-typescript/src/funcs';

const sdk = new SDK();

async function run() {
  const stations = await getStations(sdk, { search: 'Berlin' });
  if (!stations.ok) throw stations.error;

  const booking = await createBooking(sdk, {
    tripId: 'ea399ba1-6d95-433f-92d1-83f67b775594',
    passengerName: 'Alex Rider',
    hasBicycle: false,
    hasDog: true,
  });
  if (!booking.ok) throw booking.error;

  console.log(booking.value);
}

run();
```

## Automate in CI

Use the Speakeasy [sdk-generation-action](https://github.com/speakeasy-api/sdk-generation-action) to regenerate SDKs when the OpenAPI document changes.

## Summary

The NestJS app now emits an OpenAPI 3.2 document for a train travel domain and is ready for SDK generation. The key parts are:

- Configure `@nestjs/swagger` with `setOpenAPIVersion('3.2.0')`.
- Use DTOs and entity comments to generate strong schemas and parameters.
- Set tags, `operationId`, security, and retries for SDK quality.
- Write `openapi.yaml` and generate SDKs from the produced document.
