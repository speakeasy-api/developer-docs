# Components in OpenAPI

import { Table } from "@/mdx/components";

The Components object in OpenAPI are reusable bits of OpenAPI description, which can then be [referenced](/openapi/references). Reusing components can mean smaller file sizes, reduces conflicts, and improves consistency across the API. Components can even be shared them across multiple documents, allowing for improved reuse between multiple APIs.

## Components Object

The Components Object is a container for various types of reusable objects. It is a map of component names to component definitions. The Components Object can contain the following fields:

```yaml
components:
  schemas:
    MySchema:
      type: object
      properties:
        name:
          type: string
  securitySchemes:
    MySecurityScheme:
      type: http
      scheme: basic
```
