The `status` command checks the status of a deployment in a Gram project. When
no deployment ID is provided, the command shows the status of the latest
deployment.

<Callout type="info">
    The `gram status` command requires an API key and project slug to be
    specified. For more information on these values, see the [Global CLI
    Options](/docs/gram/command-line#global-options).
</Callout>

## Usage

Check the status of the latest deployment:

```bash
gram status --api-key YOUR_API_KEY --project my-project
```

Check the status of a specific deployment:

```bash
gram status --id deployment-123 --api-key YOUR_API_KEY --project my-project
```

Get deployment status as JSON:

```bash
gram status --json --api-key YOUR_API_KEY --project my-project
```

## Options

### <code>id</code>

The deployment ID to check status for. If not provided, the command shows the
status of the latest deployment.

```bash
gram status --id deployment-123 [other options]
```

### <code>json</code>

Output the deployment status as JSON. This is useful for scripting and
automation.

```bash
gram status --json [other options]
```
