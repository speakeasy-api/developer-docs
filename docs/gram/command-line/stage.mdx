The `stage` command adds sources to a deployment config that can later be passed
using `gram push`. These sources may include Gram Functions zip files, or OpenAPI
specifications.

`stage` is a parent command that contains the following subcommands:

- [**`gram stage function`**](/docs/gram/command-line/stage-function) - Stage a
  Gram Function zip file.
- [**`gram stage openapi`**](/docs/gram/command-line/stage-openapi) - Stage an
  OpenAPI document.

## Usage

```bash
gram stage [subcommand options]
```

## Options

### <code>config</code>

The path to the deployment config file. Its default value is `gram.json` in the
current working directory.

```bash
gram stage --config ./path/to/my-gram-config.json [command options]
```
